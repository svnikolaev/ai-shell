# ai-shell

**ai-shell** — это умный помощник в терминале, который по запросу на русском языке генерирует готовую bash-команду. Больше не нужно гуглить `tar -xzf` или вспоминать синтаксис `find` — просто спросите по‑русски, и утилита покажет нужную команду. Можно также попросить объяснение, чтобы разобраться, как она работает.

## Возможности

- 🧠 **Генерация команд** через LLM (OpenRouter, локальная Ollama и другие OpenAI‑совместимые бэкенды).
- 🔁 **Кэширование** повторяющихся запросов — экономит время и токены.
- 📖 **Режим объяснения** (`-e`) — показывает не только команду, но и её описание на русском.
- 🔙 **Повтор последней команды** (`-l`) — удобно для использования в пайпах (`ai -l | sh`).
- 🚫 **Стоп‑лист** — блокирует выполнение опасных команд (например, `rm -rf /`).
- 🔍 **Диагностика** (`--test`) — проверяет конфигурацию и доступность бэкендов.
- ⚙️ **Гибкая настройка** через конфиг в `~/.config/ai-shell/config.toml`.
- 🦀 **Написан на Rust** — один бинарник без зависимостей, быстрый и надёжный.

## Установка

### Из исходников (рекомендуется)

```bash
git clone https://github.com/yourusername/ai-shell.git
cd ai-shell
cargo build --release
sudo cp target/release/ai-shell /usr/local/bin/ai
```

### С помощью Cargo

```bash
cargo install --git https://github.com/yourusername/ai-shell.git
```

После установки проверьте:

```bash
ai --help
```

## Конфигурация

Первый запуск потребует создания конфигурационного файла. Утилита ищет его в стандартном месте:  
`~/.config/ai-shell/config.toml`

Создайте директорию и файл:

```bash
mkdir -p ~/.config/ai-shell
touch ~/.config/ai-shell/config.toml
```

Минимальный пример конфига с двумя бэкендами (основной OpenRouter, запасной локальная Ollama):

```toml
# Основной бэкенд — OpenRouter (бесплатные модели)
[[backends]]
api_url = "https://openrouter.ai/api/v1/chat/completions"
api_key = "sk-or-v1-..."          # ваш ключ OpenRouter
model = "deepseek/deepseek-r1:free"
timeout_secs = 30

# План Б — локальная Ollama (должен быть запущен сервер)
[[backends]]
api_url = "http://localhost:11434/v1/chat/completions"
api_key = ""
model = "qwen2.5:7b"
timeout_secs = 10

# Язык объяснений (поддерживается 'ru' и 'en')
explain_language = "ru"

# Стоп-лист: команды, содержащие эти подстроки, будут заблокированы
stop_list = [
    "rm -rf /",
    "rm -rf /*",
    "dd if=/dev/zero of=/dev/sd",
    ":(){ :|:& };:",
    "chmod -R 777 /",
    "> /dev/sda",
    "mv /* /dev/null",
    "mkfs",
    "fdisk",
]
```

Поле `api_key` можно не указывать в файле, а задать через переменную окружения `AI_API_KEY` — она имеет приоритет.

### Где взять API-ключ OpenRouter

1. Перейдите на [openrouter.ai](https://openrouter.ai) и зарегистрируйтесь.
2. В меню профиля выберите **Keys** (или откройте [openrouter.ai/keys](https://openrouter.ai/keys)).
3. Нажмите **Create API Key**, дайте имя (например, "ai-shell") и скопируйте полученный ключ.
4. Для работы с бесплатными моделями (например, `deepseek/deepseek-r1:free`) включите опцию **Enable training and logging** в [Privacy Settings](https://openrouter.ai/settings/privacy).

## Использование

```bash
ai "распакуй archive.tar.gz"
# выведет: tar -xzf archive.tar.gz

ai -e "найди все файлы больше 100М"
# выведет:
# find . -type f -size +100M
#
# Объяснение: Ищет в текущем каталоге (.) все обычные файлы (-type f) размером больше 100 мегабайт (+100M) и выводит их пути.

ai -l
# выведет последнюю сгенерированную команду

ai -l | sh
# выполнит последнюю команду (если она не в стоп-листе)
```

### Флаги

| Флаг | Альтернатива | Назначение |
|------|--------------|------------|
| `-e` | `--explain`  | Показать команду и её объяснение |
| `-l` | `--last`     | Вывести последнюю команду (из кэша) |
| `-c` | `--no-cache` | Игнорировать кэш, принудительно обратиться к LLM |
| `--test` | —        | Проверить конфигурацию и доступность бэкендов |
| `-h` | `--help`     | Показать справку |

Если не указан ни один флаг и передан вопрос, утилита выведет только команду.

## Диагностика проблем

Если утилита не работает, выполните:

```bash
ai --test
```

Эта команда проверит:
- Существование и доступность кэш-директории.
- Количество записей в стоп-листе.
- Каждый бэкенд из конфига — отправит простой тестовый запрос и покажет результат.

Пример вывода:

```
🔍 Тестирование конфигурации ai-shell

📁 Кэш-директория: /home/user/.cache/ai-shell
   ✅ доступна

🛑 Стоп-лист: 9 записей

🌐 Проверка бэкендов:
   1. https://openrouter.ai/api/v1/chat/completions ... ✅ работает
      Пример ответа: команда = "echo test", объяснение = "Выводит 'test' на экран"
   2. http://localhost:11434/v1/chat/completions ... ❌ ошибка: Ошибка соединения: connection refused
```

Это поможет быстро выявить проблемы с сетью, ключами или локальными сервисами.

## Как это работает

1. Вы вводите запрос на русском языке.
2. Утилита проверяет кэш (если не отключено) — возможно, ответ уже был получен ранее.
3. Если ответа в кэше нет, последовательно опрашиваются бэкенды из конфига (первый доступный возвращает результат).
4. Полученная команда проверяется по стоп‑листу. При совпадении выполнение прерывается с ошибкой.
5. Команда сохраняется в кэш и в файл `last_command` для последующего вызова через `-l`.
6. В зависимости от флагов выводится либо только команда, либо команда с объяснением.

**Важно:** утилита **никогда не выполняет команду автоматически**. Запуск (например, через `| sh`) — полностью ваша ответственность.

## Требования

- Linux (возможна работа и на других Unix‑подобных ОС, но тестировалась только на Linux).
- Для работы с OpenRouter нужен действительный API‑ключ.
- Для локального бэкенда необходим запущенный сервер Ollama с загруженной моделью.

## Разработка и тестирование

Проект написан на Rust и содержит как юнит-тесты, так и интеграционные тесты.

### Запуск тестов

```bash
cargo test
```

- **Юнит-тесты** находятся в каждом файле исходного кода (например, в `src/shell.rs` тестируется функция `is_dangerous`).
- **Интеграционные тесты** расположены в директории `tests/` и проверяют работу CLI (флаги `--help`, `--test` и т.д.).

Для изолированного тестирования используется `tempfile` для временных файлов кэша и конфигурации.

### Сборка

```bash
cargo build --release
```

Бинарник будет в `target/release/ai-shell`.

## Зависимости

Основные крейты:
- `clap` — парсинг аргументов командной строки.
- `ureq` — синхронные HTTP‑запросы.
- `serde` / `serde_json` — работа с JSON и сериализация.
- `toml` — чтение конфигурационных файлов.
- `dirs` — определение стандартных директорий.
- `blake3` — хэширование запросов для кэша.
- `anyhow` — удобная обработка ошибок.

Для тестирования используются `tempfile`, `assert_cmd` и `predicates` (опционально).
