# ai-shell

**ai-shell** — утилита для генерации bash-команд по запросу на естественном языке. Позволяет получать команду и её объяснение, кэширует результаты, проверяет опасные команды по стоп‑листу, ведёт историю запросов.

## Возможности

- Генерация команд через LLM (RouterAI, GPTunneL, OpenRouter, локальная Ollama и другие OpenAI‑совместимые бэкенды).
- Кэширование повторяющихся запросов.
- Режим объяснения (`-e`) — команда и описание на русском.
- Повтор последней команды — `ai` без аргументов или `ai -l`.
- Просмотр истории запросов (`-i [N]`).
- Стоп‑лист (настраиваемый) для блокировки опасных команд.
- Диагностика (`--test`) — проверка конфигурации и доступности бэкендов.
- Конфигурация в `~/.config/ai-shell/config.toml`.
- Один бинарник без зависимостей (Rust).

## Установка

### Из исходников

```bash
git clone https://github.com/yourusername/ai-shell.git
cd ai-shell
cargo build --release
sudo cp target/release/ai-shell /usr/local/bin/ai
```

### Через cargo

```bash
cargo install --git https://github.com/yourusername/ai-shell.git
```

После установки проверьте:

```bash
ai --help
```

## Конфигурация

Файл конфигурации: `~/.config/ai-shell/config.toml`.

Поля конфигурации:

- `explain_language` — язык объяснений (`ru` или `en`).
- `stop_list` — массив строк (опасные паттерны команд). Если команда содержит любой из этих паттернов (регистронезависимо), она блокируется.
- `os` — (опционально) целевая операционная система, например `"Linux"`. Если не указано, определяется автоматически.
- `backends` — массив бэкендов. Опрашиваются по порядку, пока один не вернёт успешный ответ.

Каждый бэкенд содержит:

- `api_url` — полный URL эндпоинта.
- `api_key` — (опционально) ключ API. Можно не указывать, тогда для первого бэкенда ключ можно передать через переменную окружения `AI_API_KEY`. Для локальной ollama указать `""`.
- `model` — идентификатор модели.
- `timeout_secs` — таймаут запроса в секундах (по умолчанию 30).
- `type` — (пока зарезервировано, всегда `"openai"`).

### Переменные окружения

- `AI_API_KEY` — если задана, подставляется в качестве `api_key` для **первого** бэкенда (только если у него не указан свой ключ).

### Настройка бэкендов

Ниже приведены примеры для популярных сервисов.

#### RouterAI (routerai.ru)

```toml
[[backends]]
api_url = "https://routerai.ru/api/v1/chat/completions"
api_key = "ваш_ключ"
model = "gpt-3.5-turbo"  # или другая модель
timeout_secs = 30
```

**Где взять ключ:**

1. Зарегистрируйтесь на [routerai.ru](https://routerai.ru).
2. В личном кабинете создайте API-ключ.
Список моделей доступен в документации сервиса.

#### GPTunneL (российский сервис, оплата в рублях)

```toml
[[backends]]
api_url = "https://gptunnel.ru/v1/chat/completions"
api_key = "ваш_ключ"
model = "gpt-3.5-turbo"  # или другая доступная модель
timeout_secs = 30
```

**Где взять ключ:**

1. Зарегистрируйтесь на [gptunnel.ru](https://gptunnel.ru).
2. Пополните баланс (минимум 50 ₽).
3. В личном кабинете создайте API-ключ.
Список доступных моделей смотрите на сайте.

#### OpenRouter

```toml
[[backends]]
api_url = "https://openrouter.ai/api/v1/chat/completions"
api_key = "sk-or-v1-..."  # ваш ключ
model = "nvidia/nemotron-nano-12b-v2-vl:free"
timeout_secs = 30
```

**Где взять ключ:**

1. Зарегистрируйтесь на [openrouter.ai](https://openrouter.ai).
2. Перейдите в раздел [Keys](https://openrouter.ai/keys), создайте ключ.
3. Для бесплатных моделей включите опцию **Enable training and logging** в [Privacy Settings](https://openrouter.ai/settings/privacy).

#### Локальная Ollama (с OpenAI‑совместимым эндпоинтом)

Для работы с Ollama необходимо, чтобы сервер был запущен и поддерживал эндпоинт `/v1/chat/completions` (начиная с версии 0.1.34 это включено по умолчанию).

```toml
[[backends]]
api_url = "http://localhost:11434/v1/chat/completions"
api_key = ""           # можно оставить пустым
model = "llama3.2:3b"  # любая установленная модель
timeout_secs = 10
```

**Важно:** убедитесь, что сервер Ollama запущен (`ollama serve`) и нужная модель загружена (`ollama pull llama3.2:3b`).

### Несколько бэкендов

Можно указать несколько бэкендов — они будут опрашиваться по очереди до первого успешного. Например:

```toml
[[backends]]
api_url = "https://routerai.ru/api/v1/chat/completions"
api_key = "ваш_ключ"
model = "gpt-3.5-turbo"
timeout_secs = 30

[[backends]]
api_url = "http://localhost:11434/v1/chat/completions"
api_key = ""
model = "llama3.2:3b"
timeout_secs = 10
```

### Стоп-лист

По умолчанию стоп-лист пуст (выводится предупреждение при `--test`). Чтобы активировать защиту, добавьте в конфиг:

```toml
stop_list = [
    "rm -rf /",
    "rm -rf /*",
    "dd if=/dev/zero of=/dev/sd",
    ":(){ :|:& };:",
    "chmod -R 777 /",
    "> /dev/sda",
    "mv /* /dev/null",
    "mkfs",
    "fdisk",
]
```

Проверка регистронезависимая: если команда содержит подстроку из списка, она блокируется.

### Пример общей конфигурации

Разделы explain_language и stop_list должны идти в начале конфигурации.

`config.toml`:

```toml
# Язык
explain_language = "ru"

# Стоп-лист
stop_list = [
    "rm -rf /",
    "rm -rf /*",
    "dd if=/dev/zero of=/dev/sd",
    ":(){ :|:& };:",
    "chmod -R 777 /",
    "> /dev/sda",
    "mv /* /dev/null",
    "mkfs",
    "fdisk",
]

# Основной бэкенд
[[backends]]
api_url = "https://routerai.ru/api/v1/chat/completions"
api_key = "top-secret-key"  # использовать свой api_key
model = "meta-llama/llama-4-maverick"
timeout_secs = 60

# Запасной бэкенд
[[backends]]
api_url = "https://gptunnel.ru/v1/chat/completions"
api_key = "top-secret-key"  # использовать свой api_key
model = "gpt-4.1-nano"
timeout_secs = 60

# Локальный бэкенд
[[backends]]
api_url = "http://localhost:11434/v1/chat/completions"
api_key = ""
model = "llama3.2:3b"
timeout_secs = 60
```

## Использование

```bash
# Получить команду
ai "распакуй archive.tar.gz"

# Команда + объяснение
ai -e "найди все файлы больше 100М"

# Последняя команда
ai -l

# Выполнить последнюю команду (осторожно!)
ai -l | sh

# Без аргументов — показать последнюю команду (аналогично ai -l)
ai

# Передать запрос через pipe
echo "найди все файлы больше 100М" | ai

# Объяснить готовую команду, переданную через пайп (с флагом -e)
echo "tar -xzf archive.tar.gz" | ai -e

# Просмотр истории (последние 10 запросов)
ai -i

# Последние 20 запросов с объяснениями
ai -i 20 --explain
```

### Примеры запросов

Вот несколько эффектных примеров, которые показывают, как `ai-shell` ускоряет повседневные задачи:

```bash
# 🔥 Работа с файлами
ai сожми папку src в архив с максимальным сжатием
zip -9 -r src.zip src

# 🧹 Очистка системы
ai "найди все файлы *.log в системе старше 30 дней"
find / -type f -name '*.log' -mtime +30

# 🐳 Docker
ai "запусти контейнер postgres с паролем admin и пробросом порта 5433"
docker run -e POSTGRES_PASSWORD=admin -p 5433:5432 postgres

# 📊 Мониторинг
ai "покажи процессы, которые потребляют больше всего памяти, в реальном времени"
top -o mem

# 🔄 Git
ai "отмени последний коммит, но оставь изменения в рабочей директории"
git reset --soft HEAD~1

# ⚙️ Пакетное переименование
ai "переименуй все файлы .jpeg в .jpg в текущей папке"
for file in *.jpeg; do mv "$file" "${file%.jpeg}.jpg"; done

# 🛡️ Безопасность
ai "сгенерируй сложный пароль длиной 20 символов"
openssl rand -base64 20 | tr -d '=+' | cut -c1-20

# 📝 Объяснение существующей команды
echo "rsync -av --delete /source/ /dest/" | ai -e
rsync -av --delete /source/ /dest/

Объяснение: Синхронизация директорий /source/ и /dest/ с удалением файлов в /dest/, отсутствующих в /source/

# 🚀 Многошаговая операция
ai "скачай архив с гитхаба, распакуй и перейди в распакованную папку"
curl -L https://github.com/user/repo/archive/refs/heads/main.zip -o repo.zip && unzip repo.zip && cd repo-main
```

### Флаги

| Флаг          | Назначение                                                   |
|---------------|--------------------------------------------------------------|
| `-e`          | Показать команду и объяснение                                |
| `-l`          | Вывести последнюю команду                                    |
| `-c`          | Игнорировать кэш                                             |
| `-i [N]`      | Показать последние N записей истории (по умолчанию 10)       |
| `--test`      | Диагностика конфигурации и бэкендов                          |
| `-h`          | Справка                                                      |

## Диагностика

```bash
ai --test
```

Проверяет:

- наличие и метаданные конфига;
- доступность кэш-директории;
- целевую ОС;
- стоп-лист (если поле есть в файле, но пусто — выдаётся предупреждение);
- каждый бэкенд (выполняется тестовый запрос).

## Как это работает

1. Пользователь вводит запрос или передаёт его через пайп.
2. Если ответ есть в кэше — возвращается из кэша.
3. Иначе последовательно опрашиваются бэкенды из конфига.
4. Полученная команда проверяется по стоп‑листу.
5. Команда (и объяснение) сохраняется в кэш, в `last_command` и в историю.
6. Вывод зависит от флагов.

**Важно:** ai-shell не выполняет команды автоматически.

### Пример использования

```shell
$ ai отключить репозиторий dnf sublime-text
dnf config-manager --set-disabled sublime-text

$ ai | sh
Ошибка: Эту команду нужно запускать с привилегиями суперпользователя (на большинстве систем - под именем пользователя root).

$ echo sudo $(ai) | sh
(завершено успешно)
```

## Автоматизация сборки (Makefile)

Доступные цели:

- `make test` — запустить тесты.
- `make build` — собрать release.
- `make install` — установить в `~/.cargo/bin/ai-shell`.
- `make link` — создать символическую ссылку с именем `LINK_NAME` (по умолчанию `ai`).
- `make all` (или просто `make`) — test, build, install, link.
- `make clean` — очистить.
- `make uninstall` — удалить бинарник и ссылку.

Переопределить имя ссылки:

```bash
make LINK_NAME=AI  # ссылка AI
make install link LINK_NAME=myai  # ссылка myai
```
